Section 1 — Security Architecture

1.1 Environment Variables

Exactly three variables are permitted. No AI-provider secret exists or is needed. Pollinations.ai is keyless by design — every endpoint is public HTTP with no Authorization header, no Bearer token, and no API key parameter.
VariableScopeUsage
NEXT_PUBLIC_SUPABASE_URL client-safe Supabase project URL. Safe to expose; used by browser-side Supabase client to reach the REST/Auth API.
NEXT_PUBLIC_SUPABASE_ANON_KEY client-safe Anon/public JWT. Enforces RLS on every query. Never grants elevated access — safe to ship to the browser.
SUPABASE_SERVICE_ROLE_KEY server-only Bypasses RLS for trusted server writes (shot/image persistence after AI generation). Used exclusively in Server Actions and API routes. Never referenced in any NEXT_PUBLIC_ variable or client component.

1.2 AI Layer — Zero-Secret Confirmation

  • Pollinations.ai text endpoint: https://text.pollinations.ai/openai — plain POST, no headers beyond Content-Type: application/json.
  • Pollinations.ai image endpoint: https://image.pollinations.ai/prompt/{encoded_prompt} — plain GET, no headers.
  • No Authorization, X-API-Key, or any credential header is added at any call site.
  • Both endpoints are proxied through server-side API routes to eliminate CORS exposure and centralise retry logic — not for secret injection.

1.3 Input Sanitization Policy

FieldSanitization Rule
Scene description / scriptServer-side: trim whitespace, enforce max 2000 chars, strip control characters via regex /[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g. Never interpolated raw into SQL (parameterised queries only).
Auth emailDelegate entirely to Supabase Auth. No custom parsing — Supabase validates RFC 5321 format server-side.
Auth passwordDelegate entirely to Supabase Auth. Minimum length enforced in Supabase dashboard (≥ 8 chars). Never stored or logged by application code.
Project / scene titleTrim, max 200 chars, strip HTML tags via title.replace(/<[^>]*>/g, '').
shot_id / scene_id / project_id (URL params)Validate as UUID v4 format before any DB query: /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i.

1.4 API Route Authorization Strategy

Every API route follows this exact guard pattern before any business logic executes:

// Shared helper: lib/auth-guard.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
import { NextRequest } from 'next/server'

export async function requireAuth(request: NextRequest) {
  const cookieStore = await cookies()
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll() } }
  )
  const { data: { user }, error } = await supabase.auth.getUser()
  if (error || !user) {
    return { user: null, error: 'UNAUTHORIZED' } as const
  }
  return { user, error: null } as const
}

Routes that write data use the anon client for auth check, then a separate service-role client for the write — ensuring RLS ownership is enforced at read time while the server maintains write authority.

1.5 Rate-Limiting Strategy (Vercel Hobby Tier)

Vercel Hobby does not support Edge Middleware rate-limiting via KV. The following strategy uses Supabase itself as the rate-limit store.
  • Table: rate_limits with columns (user_id uuid, route text, window_start timestamptz, count int). Upsert on each request; reject if count > threshold within the window.
  • Thresholds: generate-shots → 20 requests / 60 s per user. generate-image → 30 requests / 60 s per user.
  • Window reset: On upsert, if now() - window_start > 60s, reset count = 1 and window_start = now(); otherwise increment.
  • Response: HTTP 429 with Retry-After header set to remaining window seconds.
  • Auth forms: Supabase Auth has built-in brute-force protection (PKCE + exponential backoff). No custom layer needed.

Section 3 — API Routes: Full TypeScript Source

3.1 app/api/generate-shots/route.ts

import { NextRequest, NextResponse } from 'next/server'
import { createServerClient } from '@supabase/ssr'
import { createClient } from '@supabase/supabase-js'
import { cookies } from 'next/headers'

// ── Canonical shot type (internal — includes visual_prompt) ──
interface ShotInternal {
  shot_number: number
  angle: string
  framing: string
  movement: string
  mood: string
  visual_prompt: string
}

// ── Public response type — visual_prompt omitted ─────────────
type ShotPublic = Omit<ShotInternal, 'visual_prompt'> & {
  id: string
  scene_id: string
  order_index: number
  image_url: string | null
}

interface ContinuityContext {
  location: string
  time_of_day: string
  lighting_direction: string
}

interface RequestBody {
  scene_id: string
  script: string
  continuity_context?: ContinuityContext[]
}

const STYLE_LOCK =
  'monochrome pencil-sketch storyboard style, consistent linework, cinematic framing'

const UUID_RE =
  /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i

const REQUIRED_FIELDS: (keyof ShotInternal)[] = [
  'shot_number', 'angle', 'framing', 'movement', 'mood', 'visual_prompt',
]

function sanitizeScript(raw: string): string {
  return raw
    .trim()
    .slice(0, 2000)
    .replace(/[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g, '')
}

function extractJson(raw: string): unknown[] | null {
  const start = raw.indexOf('[')
  const end = raw.lastIndexOf(']')
  if (start === -1 || end === -1 || end <= start) return null
  try {
    const parsed = JSON.parse(raw.slice(start, end + 1))
    return Array.isArray(parsed) ? parsed : null
  } catch {
    return null
  }
}

function validateShot(obj: unknown): obj is ShotInternal {
  if (typeof obj !== 'object' || obj === null) return false
  const record = obj as Record<string, unknown>
  return REQUIRED_FIELDS.every(
    (f) => f in record && typeof record[f] === (f === 'shot_number' ? 'number' : 'string')
  )
}

function buildPrompt(script: string, context: ContinuityContext[]): string {
  const continuityBlock =
    context.length > 0
      ? `\n\nContinuity constraints from prior shots (hold constant unless the script explicitly changes them):\n${JSON.stringify(context, null, 2)}`
      : ''

  return `You are a professional storyboard supervisor. Convert the following scene script into exactly 4 to 6 shots.

Return ONLY a valid JSON array. No markdown, no explanation, no wrapper text.

Each object in the array must have exactly these fields:
- shot_number (integer, starting at 1)
- angle (string, e.g. "close-up", "wide", "dutch")
- framing (string, e.g. "over-the-shoulder", "two-shot")
- movement (string, e.g. "static", "dolly-in", "pan-left")
- mood (string, one or two words)
- visual_prompt (string — a full image generation prompt that ends with this exact suffix: "${STYLE_LOCK}")
${continuityBlock}

Scene script:
${script}`
}

async function callPollinations(prompt: string): Promise<string> {
  const response = await fetch('https://text.pollinations.ai/openai', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: 'openai',
      messages: [{ role: 'user', content: prompt }],
      jsonMode: true,
    }),
  })
  if (!response.ok) throw new Error(`Pollinations text HTTP ${response.status}`)
  const data = await response.json() as { choices?: { message?: { content?: string } }[] }
  return data?.choices?.[0]?.message?.content ?? ''
}

export async function POST(request: NextRequest): Promise<NextResponse> {
  // ── Auth guard ────────────────────────────────────────────
  const cookieStore = await cookies()
  const anonClient = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll() } }
  )
  const { data: { user } } = await anonClient.auth.getUser()
  if (!user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  // ── Rate limit check ──────────────────────────────────────
  const { data: rl } = await anonClient
    .from('rate_limits')
    .select('window_start, count')
    .eq('user_id', user.id)
    .eq('route', 'generate-shots')
    .single()

  const now = Date.now()
  const windowMs = 60_000
  const threshold = 20

  if (rl) {
    const windowAge = now - new Date(rl.window_start).getTime()
    if (windowAge < windowMs && rl.count >= threshold) {
      const retryAfter = Math.ceil((windowMs - windowAge) / 1000)
      return NextResponse.json(
        { error: 'Rate limit exceeded' },
        { status: 429, headers: { 'Retry-After': String(retryAfter) } }
      )
    }
  }

  // ── Parse and validate request body ──────────────────────
  let body: RequestBody
  try {
    body = (await request.json()) as RequestBody
  } catch {
    return NextResponse.json({ error: 'Invalid JSON body' }, { status: 400 })
  }

  const { scene_id, script, continuity_context = [] } = body

  if (!scene_id || !UUID_RE.test(scene_id)) {
    return NextResponse.json({ error: 'Invalid scene_id' }, { status: 400 })
  }
  if (!script || typeof script !== 'string') {
    return NextResponse.json({ error: 'Missing script' }, { status: 400 })
  }

  const cleanScript = sanitizeScript(script)

  // ── Ownership check — user must own the parent scene ─────
  const { data: sceneOwner } = await anonClient
    .from('scenes')
    .select('id')
    .eq('id', scene_id)
    .single()

  if (!sceneOwner) {
    return NextResponse.json({ error: 'Scene not found' }, { status: 404 })
  }

  // ── Call Pollinations with one retry on JSON parse failure
  const prompt = buildPrompt(cleanScript, continuity_context)
  let shots: ShotInternal[] | null = null

  for (let attempt = 0; attempt < 2; attempt++) {
    let raw: string
    try {
      raw = await callPollinations(prompt)
    } catch {
      if (attempt === 1) {
        return NextResponse.json({ error: 'AI service unavailable' }, { status: 502 })
      }
      continue
    }
    const parsed = extractJson(raw)
    if (parsed && parsed.every(validateShot)) {
      shots = parsed as ShotInternal[]
      break
    }
    if (attempt === 1) {
      return NextResponse.json({ error: 'AI returned unparseable output' }, { status: 502 })
    }
  }

  if (!shots) {
    return NextResponse.json({ error: 'Shot generation failed' }, { status: 502 })
  }

  // ── Append style-lock suffix to every visual_prompt ──────
  const lockedShots = shots.map((s) => ({
    ...s,
    visual_prompt: s.visual_prompt.endsWith(STYLE_LOCK)
      ? s.visual_prompt
      : `${s.visual_prompt.trimEnd()}, ${STYLE_LOCK}`,
  }))

  // ── Get current max order_index for this scene ────────────
  const { data: existingShots } = await anonClient
    .from('shots')
    .select('order_index')
    .eq('scene_id', scene_id)
    .order('order_index', { ascending: false })
    .limit(1)

  const baseIndex = existingShots?.[0]?.order_index ?? -1

  // ── Persist using service-role client ─────────────────────
  const serviceClient = createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!
  )

  const rows = lockedShots.map((s, i) => ({
    scene_id,
    shot_number: s.shot_number,
    angle: s.angle,
    framing: s.framing,
    movement: s.movement,
    mood: s.mood,
    visual_prompt: s.visual_prompt,
    order_index: baseIndex + 1 + i,
    image_url: null,
  }))

  const { data: inserted, error: insertError } = await serviceClient
    .from('shots')
    .insert(rows)
    .select('id, scene_id, shot_number, angle, framing, movement, mood, order_index, image_url')

  if (insertError || !inserted) {
    return NextResponse.json({ error: 'Database write failed' }, { status: 500 })
  }

  // ── Update rate limit counter ─────────────────────────────
  if (rl && now - new Date(rl.window_start).getTime() < windowMs) {
    await anonClient
      .from('rate_limits')
      .update({ count: rl.count + 1 })
      .eq('user_id', user.id)
      .eq('route', 'generate-shots')
  } else {
    await anonClient
      .from('rate_limits')
      .upsert({ user_id: user.id, route: 'generate-shots', window_start: new Date().toISOString(), count: 1 })
  }

  // ── Return public shape — visual_prompt excluded ──────────
  const publicShots: ShotPublic[] = inserted
  return NextResponse.json({ shots: publicShots }, { status: 200 })
}

3.2 app/api/generate-image/route.ts

import { NextRequest, NextResponse } from 'next/server'
import { createServerClient } from '@supabase/ssr'
import { createClient } from '@supabase/supabase-js'
import { cookies } from 'next/headers'

const UUID_RE =
  /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i

interface RequestBody {
  shot_id: string
}

async function fetchImageWithRetry(url: string): Promise<Response> {
  for (let attempt = 0; attempt < 2; attempt++) {
    try {
      const res = await fetch(url)
      if (res.ok) return res
      if (attempt === 1) throw new Error(`HTTP ${res.status}`)
    } catch (err) {
      if (attempt === 1) throw err
    }
  }
  // unreachable — loop always throws or returns on attempt 1
  throw new Error('Image fetch failed')
}

export async function POST(request: NextRequest): Promise<NextResponse> {
  // ── Auth guard ────────────────────────────────────────────
  const cookieStore = await cookies()
  const anonClient = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll() } }
  )
  const { data: { user } } = await anonClient.auth.getUser()
  if (!user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  // ── Rate limit check ──────────────────────────────────────
  const { data: rl } = await anonClient
    .from('rate_limits')
    .select('window_start, count')
    .eq('user_id', user.id)
    .eq('route', 'generate-image')
    .single()

  const now = Date.now()
  const windowMs = 60_000
  const threshold = 30

  if (rl) {
    const windowAge = now - new Date(rl.window_start).getTime()
    if (windowAge < windowMs && rl.count >= threshold) {
      const retryAfter = Math.ceil((windowMs - windowAge) / 1000)
      return NextResponse.json(
        { error: 'Rate limit exceeded' },
        { status: 429, headers: { 'Retry-After': String(retryAfter) } }
      )
    }
  }

  // ── Parse and validate request body ──────────────────────
  let body: RequestBody
  try {
    body = (await request.json()) as RequestBody
  } catch {
    return NextResponse.json({ error: 'Invalid JSON body' }, { status: 400 })
  }

  const { shot_id } = body

  if (!shot_id || !UUID_RE.test(shot_id)) {
    return NextResponse.json({ error: 'Invalid shot_id' }, { status: 400 })
  }

  // ── Fetch visual_prompt — ownership enforced by RLS ───────
  const { data: shot } = await anonClient
    .from('shots')
    .select('id, visual_prompt')
    .eq('id', shot_id)
    .single()

  if (!shot) {
    return NextResponse.json({ error: 'Shot not found' }, { status: 404 })
  }

  // ── Build Pollinations image URL (no API key) ─────────────
  const encodedPrompt = encodeURIComponent(shot.visual_prompt)
  const imageUrl = `https://image.pollinations.ai/prompt/${encodedPrompt}?model=flux&width=768&height=432&nologo=true`

  // ── Fetch image server-side with single retry ─────────────
  let imageBuffer: ArrayBuffer
  let contentType: string
  try {
    const imageRes = await fetchImageWithRetry(imageUrl)
    imageBuffer = await imageRes.arrayBuffer()
    contentType = imageRes.headers.get('content-type') ?? 'image/jpeg'
  } catch {
    // Do NOT include visual_prompt in error response
    return NextResponse.json({ error: 'Image generation failed' }, { status: 502 })
  }

  // ── Persist image as data URL via service-role client ─────
  const base64 = Buffer.from(imageBuffer).toString('base64')
  const dataUrl = `data:${contentType};base64,${base64}`

  const serviceClient = createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!
  )

  const { error: updateError } = await serviceClient
    .from('shots')
    .update({ image_url: dataUrl })
    .eq('id', shot_id)

  if (updateError) {
    return NextResponse.json({ error: 'Failed to persist image' }, { status: 500 })
  }

  // ── Update rate limit counter ─────────────────────────────
  if (rl && now - new Date(rl.window_start).getTime() < windowMs) {
    await anonClient
      .from('rate_limits')
      .update({ count: rl.count + 1 })
      .eq('user_id', user.id)
      .eq('route', 'generate-image')
  } else {
    await anonClient
      .from('rate_limits')
      .upsert({ user_id: user.id, route: 'generate-image', window_start: new Date().toISOString(), count: 1 })
  }

  // ── Stream binary image back to client ────────────────────
  return new NextResponse(imageBuffer, {
    status: 200,
    headers: { 'Content-Type': contentType },
  })
}

4.2 Data Flow Map

LayerResponsibilityWhat it passes down
page.tsx (Server Component) Fetches projects / scenes / shots from Supabase using server client (full row including visual_prompt) Passes mapped prop arrays to client components — visual_prompt stripped at this boundary via TypeScript mapped type (see §4.3)
Board.tsx (Client Component) Owns useState<ShotPublic[]>; receives initial shots as props; triggers fetch('/api/generate-shots') and fetch('/api/generate-image') Passes individual ShotPublic objects to ShotCard.tsx
ShotCard.tsx (Client Component) Renders shot metadata and image_url. Has no knowledge of visual_prompt — not in its props type. —
Server Actions (lib/actions/) Handle mutations (create/delete/reorder). Called from Client Components via "use server" functions. Each action re-validates the session internally. Return { success: boolean; error?: string }

4.3 The visual_prompt Firewall

Enforcement layer: the Server Component page, not the Client Component, is responsible for stripping visual_prompt. TypeScript makes this impossible to bypass accidentally. 
// lib/types.ts

// ── Full DB row shape (server-only) ──────────────────────────
export interface ShotRow {
  id: string
  scene_id: string
  shot_number: number
  angle: string
  framing: string
  movement: string
  mood: string
  visual_prompt: string      // present in DB row
  image_url: string | null
  order_index: number
}

// ── Client-safe shape — visual_prompt structurally absent ────
export type ShotPublic = Omit<ShotRow, 'visual_prompt'>

// ── Mapping function used in every server page that fetches shots
export function toPublicShot(row: ShotRow): ShotPublic {
  const { visual_prompt: _stripped, ...pub } = row
  return pub
}
// app/(app)/projects/[projectId]/scenes/[sceneId]/page.tsx  (excerpt)
import { toPublicShot, ShotPublic } from '@/lib/types'

// shots is ShotRow[] from DB — visual_prompt present here
const shots: ShotRow[] = await fetchShotsForScene(sceneId)

// Strip before passing to any Client Component
const publicShots: ShotPublic[] = shots.map(toPublicShot)

return <Board initialShots={publicShots} sceneId={sceneId} />

Because Board.tsx and ShotCard.tsx accept only ShotPublic, TypeScript will produce a compile error if any call site attempts to pass a ShotRow (which carries visual_prompt) directly.

4.4 Drag-and-Drop Reorder Flow

// components/board/Board.tsx (reorder excerpt)
import { DragDropContext, Droppable, DropResult } from '@hello-pangea/dnd'
import { reorderShots } from '@/lib/actions/shot-actions'

function handleDragEnd(result: DropResult) {
  if (!result.destination) return
  const reordered = Array.from(shots)
  const [moved] = reordered.splice(result.source.index, 1)
  reordered.splice(result.destination.index, 0, moved)
  // Optimistic UI update
  setShots(reordered)
  // Persist new order_index values
  void reorderShots(
    reordered.map((s, i) => ({ id: s.id, order_index: i }))
  )
}
